# Skill Routing

- Before making a non-trivial change, read `~/.codex/skill-routing.md` when it exists.
- Use that file to decide whether a local skill, project convention, or manual tool workflow should guide the edit.
- If one high-confidence route matches, apply that route's guidance before editing.
- If several complementary routes match, use the minimal useful set.
- If several routes are mutually exclusive, ask the user to choose.
- If a route points to a tool unavailable in aider, explain the missing capability and use the closest available editing workflow.

Load this file in aider with `/read CONVENTIONS.md` or `aider --read CONVENTIONS.md`.
